# <a name="connect-one-identity-safeguard"></a>Verbinden von One Identity Safeguard
Der One Identity Safeguard-Datenconnector erweitert den standardmäßigen CEF-Connector (Common Event Format) um Dashboards, die für Safeguard for Privileged Sessions spezifisch sind. Verwenden Sie diesen Connector, um die von Ihrem Gerät erzeugten Ereignisse für Visualisierungen, Warnungen, Untersuchungen usw. zu verwenden.

[!INCLUDE [reference-to-feature-availability](includes/reference-to-feature-availability.md)]

## <a name="prerequisites"></a>Voraussetzungen

- Lese- und Schreibberechtigungen für den Azure Sentinel-Arbeitsbereich.

## <a name="connect-azure-sentinel-to-one-identity-safeguard"></a>Verbinden von Azure Sentinel mit One Identity Safeguard

1. Navigieren Sie im Azure-Portal zu **Azure Sentinel** > **Datenconnectors**, und wählen Sie dann den Connector **One Identity Safeguard** aus.

2. Wählen Sie **Connectorseite öffnen** aus.

3. Befolgen Sie die Anweisungen auf der Seite **One Identity Safeguard**.
